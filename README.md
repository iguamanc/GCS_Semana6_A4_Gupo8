#App movil ParqueAndo / Grupo 8

#Roles sugeridos (Autores)
1. Auditor de Configuración (Fausto Cando)
  Responsabilidades:
- Revisar la estructura del repositorio (ramas, commits, etiquetas).
- Verificar que la configuración del proyecto (dependencias, archivos de entorno, documentación) esté completa y consistente.
- Generar un informe de auditoría con hallazgos y recomendaciones.
- Evidencia en el repositorio:
- Issues documentando hallazgos.
- Commits corrigiendo configuraciones.
- Wiki/README con checklist de auditoría.

2. Gestor de Trazabilidad y Evidencias(Luis Naranjo)
  Responsabilidades:
- Asegurar que cada cambio esté vinculado a un issue o PR.
- Documentar la trazabilidad entre requisitos, código y pruebas.
- Mantener un registro de evidencias (capturas, logs, reportes).
- Evidencia en el repositorio:
- Issues enlazados a PRs.
- Commits con mensajes claros y referenciados.
- Carpeta de evidencias (ej. /docs/evidencias).

3. Responsable de Release y Entrega (Isidoro Guaman)
  Responsabilidades:
- Preparar la versión candidata (release branch, changelog).
- Ejecutar la liberación controlada (tag, empaquetado, documentación).
- Validar integridad y calidad antes de la entrega.
- Evidencia en el repositorio:
- Pull Request de la rama de release.
- Tag con versión y changelog.
- Registro de entrega (ej. release notes).
